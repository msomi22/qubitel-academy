# Cline Response Format

## For implementation tasks

Respond with:

1. Summary
2. Files changed
3. Validation run
4. Manual checks
5. Notes / follow-ups

Keep the response direct.

Do not hide skipped validation.

## For planning tasks

Respond with:

1. Understanding
2. Relevant files/docs to inspect
3. Proposed approach
4. Risks
5. Acceptance criteria
6. Validation plan

## For UI tasks

Include:

- viewport assumptions
- mobile behavior
- desktop behavior
- accessibility notes
- screenshots needed, if applicable

## For content tasks

Include:

- content path
- manifest registration
- metadata status
- production visibility
- validation commands
