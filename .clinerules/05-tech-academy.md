# Technology Academy Rules

## Purpose

Technology Academy focuses on software engineering mastery and interview preparation.

Current areas include:

- DSA
- System Design
- Java
- Kubernetes / CKAD
- Aptitude Test Practice
- ML / AI
- Engineering Leadership

## Content style

Tech content must be practical, explanation-first, and production-aware.

For DSA:

- explain the pattern
- include reasoning, not only the answer
- use clear examples
- avoid vague prompts

For System Design:

- include requirements
- include constraints
- include trade-offs
- include failure modes
- include observability where useful
- prefer structured rich body blocks when appropriate

For Java/backend:

- be precise about language/runtime behavior
- avoid oversimplified or incorrect explanations
- include edge cases where useful

## Topic/category structure

Keep content close to the owning academy/category/topic.

Do not add new tech content to legacy banks.

Register new files in the relevant topic manifest.
